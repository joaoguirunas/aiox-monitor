# Story 1.4 — Hook Python + Instalação Automática

## Story

**Como** usuário que quer monitorar projetos Claude Code,
**Quero** executar `npm run install-hook` uma única vez,
**Para que** todos os eventos do Claude Code em qualquer projeto sejam automaticamente capturados e enviados ao aiox-monitor.

## Acceptance Criteria

- [ ] **AC1** — Arquivo `hooks/aiox-monitor-hook.py` existe e é um script Python 3 válido (sem dependências externas — só stdlib)
- [ ] **AC2** — O hook lê stdin como JSON (payload do Claude Code) sem falhar se stdin estiver vazio ou com JSON inválido
- [ ] **AC3** — O hook captura `os.getppid()` como `terminal_pid` e `os.environ.get("CLAUDE_SESSION_ID")` como `terminal_session_id`
- [ ] **AC4** — O hook captura `CLAUDE_HOOK_TYPE` do ambiente para identificar o tipo de evento
- [ ] **AC5** — O hook infere o `project_path` usando `os.environ.get("CLAUDE_PROJECT_DIR", os.getcwd())`
- [ ] **AC6** — O hook envia POST para `http://localhost:8888/api/events` com timeout de 2 segundos
- [ ] **AC7** — Falha de conexão (servidor offline) é silenciosa — hook NÃO imprime erro nem retorna exit code != 0
- [ ] **AC8** — O hook detecta o agente ativo procurando por `@dev`, `@qa`, `@architect`, etc. no conteúdo do payload
- [ ] **AC9** — `scripts/install-hook.ts` copia o hook para `~/.claude/hooks/aiox-monitor-hook.py` e configura `~/.claude/hooks/hooks.json`
- [ ] **AC10** — O script de instalação faz backup de `hooks.json` existente antes de modificar
- [ ] **AC11** — O script de instalação adiciona o hook para todos os 5 tipos de evento: `PreToolUse`, `PostToolUse`, `UserPromptSubmit`, `Stop`, `SubagentStop`
- [ ] **AC12** — `scripts/uninstall-hook.ts` remove o hook de `hooks.json` e deleta o arquivo `.py`
- [ ] **AC13** — `npm run install-hook` e `npm run uninstall-hook` executam corretamente (via `tsx scripts/install-hook.ts`)

## Dev Notes

### Estrutura do hooks.json do Claude Code
```json
{
  "hooks": {
    "PreToolUse": [
      { "command": "python3 ~/.claude/hooks/aiox-monitor-hook.py" }
    ],
    "PostToolUse": [
      { "command": "python3 ~/.claude/hooks/aiox-monitor-hook.py" }
    ],
    "UserPromptSubmit": [
      { "command": "python3 ~/.claude/hooks/aiox-monitor-hook.py" }
    ],
    "Stop": [
      { "command": "python3 ~/.claude/hooks/aiox-monitor-hook.py" }
    ],
    "SubagentStop": [
      { "command": "python3 ~/.claude/hooks/aiox-monitor-hook.py" }
    ]
  }
}
```

### Por que os.getppid() e não os.getpid()?
O hook é um subprocess chamado pelo Claude Code. O PID do hook muda a cada execução. O que identifica o terminal é o PID do processo Claude Code que invocou o hook — que é o PID pai (`getppid()`).

### CLAUDE_PROJECT_DIR
O Claude Code injeta a variável de ambiente `CLAUDE_PROJECT_DIR` com o path do projeto ativo. É o método mais confiável para identificar o projeto.

### Detecção de agente
O Claude Code pode mencionar `@dev` ou `@architect` no conteúdo dos eventos. A detecção é melhor-esforço — se não encontrar, usa `@unknown`.

### Zero dependências externas no Python
O hook deve funcionar com Python 3.6+ sem pip install. Usar apenas: `json`, `sys`, `os`, `urllib.request`, `urllib.error`.

### Arquivo hooks.json — merge não-destrutivo
O script de instalação NÃO sobrescreve hooks existentes de outras ferramentas. Adiciona apenas se o comando do aiox-monitor não está presente. O backup `.bak` protege contra corrupção.
