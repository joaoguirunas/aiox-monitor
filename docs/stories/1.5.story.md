# Story 1.5 — Endpoints de Leitura

## Story

**Como** frontend do aiox-monitor,
**Quero** consultar projetos, agentes e eventos via endpoints GET com suporte a filtros,
**Para que** os modos Kanban e Lista possam exibir dados do banco SQLite.

## Acceptance Criteria

- [x] **AC1** — `GET /api/projects` retorna lista de projetos ordenada por `last_active DESC`
- [x] **AC2** — `GET /api/projects/:id` retorna um projeto com seus agentes e terminais ativos
- [x] **AC3** — `GET /api/agents` retorna todos os agentes, aceitando `?project_id=N` como filtro
- [x] **AC4** — `GET /api/events` retorna eventos paginados (padrão: 100 por página), aceitando filtros: `?project_id=`, `?agent_id=`, `?type=`, `?terminal_id=`, `?limit=`, `?offset=`, `?since=` (ISO timestamp)
- [x] **AC5** — `GET /api/events` retorna objeto com `{ events: Event[], total: number, hasMore: boolean }`
- [x] **AC6** — `GET /api/agents` inclui campo `terminal_count` (quantidade de terminais ativos por agente)
- [x] **AC7** — `GET /api/stats` retorna: total de eventos hoje, agentes ativos, projetos ativos, evento mais recente
- [x] **AC8** — Todos os endpoints retornam `Content-Type: application/json`
- [x] **AC9** — Parâmetros inválidos (ex: `project_id=abc`) retornam `400` com mensagem clara
- [x] **AC10** — Endpoints são Server-Side apenas (Next.js Route Handlers) — sem client-side fetch direto ao SQLite

## Dev Notes

### Queries SQL com filtros opcionais (better-sqlite3)
```typescript
// Padrão para queries com filtros dinâmicos
function getEvents(filters: EventFilters) {
  const conditions: string[] = [];
  const params: unknown[] = [];

  if (filters.projectId) {
    conditions.push('project_id = ?');
    params.push(filters.projectId);
  }
  if (filters.type) {
    conditions.push('type = ?');
    params.push(filters.type);
  }
  // ...

  const where = conditions.length ? `WHERE ${conditions.join(' AND ')}` : '';
  const limit = filters.limit ?? 100;
  const offset = filters.offset ?? 0;

  const countStmt = db.prepare(`SELECT COUNT(*) as total FROM events ${where}`);
  const total = (countStmt.get(...params) as { total: number }).total;

  const stmt = db.prepare(`SELECT * FROM events ${where} ORDER BY created_at DESC LIMIT ? OFFSET ?`);
  const events = stmt.all(...params, limit, offset) as Event[];

  return { events, total, hasMore: offset + events.length < total };
}
```
