# Story 1.7 — Validação End-to-End da Fase 1

## Story

**Como** desenvolvedor do aiox-monitor,
**Quero** validar que o fluxo completo funciona — desde o hook Python capturando eventos até a visualização no Modo Lista,
**Para que** a Fase 1 possa ser considerada concluída e entregue.

## Acceptance Criteria

- [x] **AC1** — Com `npm run dev` rodando, o hook Python consegue enviar um evento e obter `{ success: true }` em resposta
- [x] **AC2** — Após instalar o hook (`npm run install-hook`), eventos reais do Claude Code aparecem no Modo Lista sem ação manual
- [x] **AC3** — Múltiplos terminais no mesmo projeto criam múltiplos terminais na tabela (distinguíveis por PID)
- [x] **AC4** — Fechar o Claude Code (evento `Stop`) marca o terminal como `inactive` no banco
- [x] **AC5** — O Modo Lista exibe projetos, agentes e tipos de eventos corretos com badges coloridos
- [x] **AC6** — Filtros no Modo Lista funcionam: filtrar por projeto mostra apenas eventos daquele projeto
- [x] **AC7** — Click em evento abre drawer com detalhes corretos (timestamp, agente, tool, resumo)
- [x] **AC8** — Navbar exibe os três modos e o link ativo está destacado
- [x] **AC9** — `npm run build` compila sem erros de TypeScript ou lint
- [x] **AC10** — Fluxo de instalação documentado no README.md com 4 comandos: clone, install, install-hook, dev

## Tasks / Subtasks

- [x] **Task 1** — Teste do endpoint com curl (AC: 1)
  ```bash
  # 1. Iniciar servidor
  npm run dev

  # 2. Enviar evento simulado
  curl -X POST http://localhost:8888/api/events \
    -H "Content-Type: application/json" \
    -d '{
      "hook_type": "PreToolUse",
      "project_path": "/Users/user/Desktop/test-project",
      "project_name": "test-project",
      "agent_name": "@dev",
      "tool_name": "Read",
      "input": {"file_path": "/src/app.tsx"},
      "terminal_pid": 99999,
      "terminal_session_id": "test-session-1"
    }'

  # Esperado: {"success":true,"event_id":1}

  # 3. Verificar no banco
  sqlite3 data/monitor.db "SELECT * FROM events;"
  sqlite3 data/monitor.db "SELECT * FROM terminals;"
  sqlite3 data/monitor.db "SELECT * FROM projects;"
  ```

- [x] **Task 2** — Teste com múltiplos terminais (AC: 3, 4)
  ```bash
  # Terminal 1 (PID 11111)
  curl -X POST http://localhost:8888/api/events \
    -d '{"hook_type":"PreToolUse","project_path":"/tmp/proj","terminal_pid":11111,"agent_name":"@dev","tool_name":"Edit"}'

  # Terminal 2 (PID 22222) no mesmo projeto
  curl -X POST http://localhost:8888/api/events \
    -d '{"hook_type":"PreToolUse","project_path":"/tmp/proj","terminal_pid":22222,"agent_name":"@qa","tool_name":"Bash"}'

  # Verificar: 2 terminais na tabela terminals para /tmp/proj
  sqlite3 data/monitor.db "SELECT * FROM terminals;"

  # Fechar terminal 1
  curl -X POST http://localhost:8888/api/events \
    -d '{"hook_type":"Stop","project_path":"/tmp/proj","terminal_pid":11111}'

  # Verificar: terminal 11111 com status=inactive
  sqlite3 data/monitor.db "SELECT pid, status FROM terminals;"
  ```
