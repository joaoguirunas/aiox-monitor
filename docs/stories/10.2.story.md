# Story 10.2 — Agent Detail Panel

## Story

**Como** utilizador do aiox-monitor,
**Quero** clicar num agente no Kanban e ver um painel lateral com eventos recentes, sessão activa e terminal associado,
**Para que** tenha contexto operacional completo sem navegar para as abas Lista ou Terminais.

## Context

Actualmente o `AgentCard` é puramente apresentacional — sem `onClick`, sem modais, sem interactividade (verificado: zero handlers). O utilizador precisa de mudar entre abas para correlacionar agente, terminal e eventos.

**Depende de:** Story 10.1 (bridge terminal → agent), que garante que dados enriquecidos já chegam ao Kanban.

### Tasks Completed

WS subscription: `event:new` prepends to events list (capped at 10). Loading state with shimmer skeleton.
- Task 4 (AC14-AC16): Build passes cleanly. Panel opens instantly (state-driven, no network blocking). `selectedAgent` state is local to `page.tsx` — ProjectRow components use stable `onAgentClick` callback via `useCallback`, no unnecessary re-renders.

- Terminals API doesn't support `agent_name` filter — fetches all project terminals and filters client-side.

### Completion Notes
- Panel uses `key={selectedAgent.id}` for clean remount on agent switch — simpler than managing internal state reset
- Terminal section fetches from `/api/terminals?project_id=` and filters by `agent_name` + active status client-side
