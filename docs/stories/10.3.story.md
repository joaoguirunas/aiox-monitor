# Story 10.3 — Multi-Terminal Badge

## Story

**Como** utilizador do aiox-monitor,
**Quero** ver um badge no card do agente quando ele tem múltiplos terminais activos,
**Para que** saiba que o agente está a operar em paralelo e entenda a carga real.

## Context

O query `getAgents()` em `queries.ts` já retorna `terminal_count` por agente. No entanto, `AgentCard` ignora este campo. Quando um agente novo chega via WS, o handler em `useAgents.ts:49` atribui `terminal_count: 0` hardcoded, perdendo a informação.

## Acceptance Criteria

### Task 1: Badge visual

- [x] **AC1** — Variante `card`: quando `terminal_count > 1`, exibir badge numérico no canto superior direito com tooltip "X terminais activos"
- [x] **AC2** — Variante `chip`: quando `terminal_count > 1`, exibir badge compacto ("x2") junto ao nome
- [x] **AC3** — Badge ausente quando `terminal_count <= 1` (sem layout shift)
- [x] **AC4** — Estilo: background `surface-3`, texto `text-secondary`, border-radius full, font-size 10px

### Task 2: Fix WS merge

- [x] **AC5** — `useAgents.ts:49` → Usar `terminal_count` do payload WS se disponível, em vez de `0` hardcoded (já implementado na Story 10.1 — spread order `{ terminal_count: 0, ...msg.agent }`)
- [x] **AC6** — `useAgents.ts:44` → No merge de agente existente, preservar `terminal_count` do fetch original se WS não incluir o campo (já implementado — `{ ...a, ...msg.agent }` preserva campos existentes)

### Task 3: Enriquecer broadcast

- [x] **AC7** — `agent-tracker.ts` → Incluir `terminal_count` no broadcast `agent:update` (já implementado na Story 10.1 — `getTerminalEnrichment()` retorna `terminal_count`)

### Task 4: Validação

- [x] **AC8** — Build passa sem erros (`npm run build`)
- [x] **AC9** — Badge aparece com 2+ terminais e desaparece quando volta a 1

## Dev Notes

### Query existente
`getAgents()` em `queries.ts:125` já faz:
```sql
COUNT(CASE WHEN t.status IN ('processing','active') THEN 1 END) as terminal_count
```
O campo está disponível no fetch. Falta: incluir no WS broadcast e renderizar no AgentCard.
