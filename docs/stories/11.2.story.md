# Story 11.2 — WebSocket PTY Namespace

## Story

**Como** desenvolvedor do AIOX Monitor,
**Quero** um namespace WebSocket `/pty` no server.ts que faça bridge entre clientes browser e processos node-pty,
**Para que** o frontend possa receber stdout em tempo real e enviar stdin aos terminais via WebSocket binário.

## Referências

- PRD: [sala-de-comando-prd.md](../prd/sala-de-comando-prd.md) — FR-1.4, FR-1.5, FR-1.8
- Arquitetura: [sala-de-comando-architecture.md](../architecture/sala-de-comando-architecture.md) — Seções 4.1–4.4, 7.1–7.2

## Acceptance Criteria

- [x] **AC1** — Novo `WebSocketServer` no path `/pty` coexiste com o `/ws` existente sem conflito
- [x] **AC2** — Conexão WS a `/pty?id={terminalId}` associa o client ao processo PTY correspondente
- [x] **AC3** — Client recebe scrollback buffer completo ao conectar (JSON text frame `{"type":"scrollback","data":"..."}`)
- [x] **AC4** — Client recebe status atual ao conectar (JSON text frame `{"type":"status","status":"..."}`)
- [x] **AC5** — stdout do processo é transmitido ao client como binary frames (Uint8Array) em < 50ms
- [x] **AC6** — Client envia stdin como binary frames que são escritos no processo PTY
- [x] **AC7** — Client envia resize como JSON text frame `{"type":"resize","cols":N,"rows":N}` que redimensiona o PTY
- [x] **AC8** — Client recebe `{"type":"exit","code":N}` quando processo termina
- [x] **AC9** — Client recebe `{"type":"error","message":"..."}` e conexão é fechada se terminalId não existe
- [x] **AC10** — Múltiplos clients podem conectar ao mesmo terminalId (broadcast)
- [x] **AC11** — Desconexão do último client não mata o processo (processo persiste para reconexão)
- [x] **AC12** — Ping/pong keep-alive funciona (`{"type":"ping"}` → `{"type":"pong"}`)
- [x] **AC13** — Upgrade handler no server.ts roteia `/ws` e `/pty` corretamente
- [x] **AC14** — Graceful shutdown: `SIGTERM`/`SIGINT` chamam `ProcessManager.killAll()`

## Dev Notes

- O server.ts atual tem ~130 linhas com upgrade handler para `/ws`
- As alterações ao server.ts são mínimas: import + criar ptyWss + adicionar branch no upgrade handler
- Binary frames para stdout (performance), JSON text frames para controle (parseable)
- `ws.binaryType = 'arraybuffer'` deve ser definido no client-side
- Mapa `clients: Map<terminalId, Set<WebSocket>>` para broadcast
- A função `broadcastToClients(terminalId, data)` é chamada pelo ProcessManager.onData
