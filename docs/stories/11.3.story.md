# Story 11.3 — API Routes Command Room

## Story

**Como** desenvolvedor do AIOX Monitor,
**Quero** API Routes REST para gerenciar processos de terminal (spawn, list, kill, resize),
**Para que** o frontend possa controlar o ciclo de vida dos terminais via HTTP antes de conectar o WebSocket.

## Referências

- PRD: [sala-de-comando-prd.md](../prd/sala-de-comando-prd.md) — FR-1.2, FR-1.6, FR-1.7, FR-1.9
- Arquitetura: [sala-de-comando-architecture.md](../architecture/sala-de-comando-architecture.md) — Seção 5.1–5.2

## Acceptance Criteria

- [ ] **AC1** — `POST /api/command-room/spawn` spawna processo Claude com `agentName` e `projectPath`, retorna 201 com `{id, agentName, pid, status, wsUrl}`
- [ ] **AC2** — `POST /api/command-room/spawn` retorna 400 se `agentName` ou `projectPath` estão ausentes
- [ ] **AC3** — `POST /api/command-room/spawn` retorna 503 se limite de 10 processos atingido
- [ ] **AC4** — `POST /api/command-room/spawn` aceita `cols`, `rows`, `initialPrompt` opcionais
- [ ] **AC5** — `GET /api/command-room/list` retorna `{terminals: [...], count: N, maxTerminals: 10}`
- [ ] **AC6** — `GET /api/command-room/[id]` retorna detalhes do processo ou 404
- [ ] **AC7** — `DELETE /api/command-room/[id]` mata o processo e retorna `{id, killed: true}` ou 404
- [ ] **AC8** — `POST /api/command-room/[id]/resize` aceita `{cols, rows}` e redimensiona o PTY, retorna 200 ou 404
- [ ] **AC9** — `POST /api/command-room/[id]/resize` retorna 400 se `cols` ou `rows` ausentes ou inválidos
- [ ] **AC10** — Todos os endpoints retornam JSON com `Content-Type: application/json`
- [ ] **AC11** — Validação de `projectPath`: retorna 400 se caminho não existe ou não é diretório

## Dev Notes

- Usar Next.js App Router API Routes (`export async function POST/GET/DELETE`)
- ProcessManager é singleton: acessar via `ProcessManager.getInstance()`
- `wsUrl` retornada pelo spawn: `/pty?id={uuid}` (o frontend usa isso para conectar o WebSocket)
- O `initialPrompt` é escrito no stdin do processo após um delay de 500ms (para o Claude inicializar)
- Todos os endpoints são locais (localhost only), sem autenticação necessária
