# Story 2.2 — useWebSocket Hook

## Story

**Como** componente React do aiox-monitor,
**Quero** um hook que gerencie a conexão WebSocket com reconexão automática,
**Para que** qualquer página possa receber atualizações em tempo real sem reimplementar a lógica de conexão.

## Acceptance Criteria

- [ ] **AC1** — `useWebSocket()` conecta em `ws://localhost:8888/ws` automaticamente ao montar
- [ ] **AC2** — Hook retorna `{ connected: boolean, lastMessage: WsMessage | null }`
- [ ] **AC3** — Ao perder conexão, tenta reconectar com backoff: 1s, 2s, 4s, 8s, 16s (máximo 5 tentativas)
- [ ] **AC4** — Após 5 falhas consecutivas, para de tentar e mantém `connected: false`
- [ ] **AC5** — Ao reconectar com sucesso, reseta contador de tentativas
- [ ] **AC6** — `useWebSocket()` faz cleanup correto (fecha socket, cancela timers) no unmount
- [ ] **AC7** — `ConnectionStatus` component (Story 1.6) passa a refletir estado real da conexão WS
- [ ] **AC8** — Hook funciona corretamente com React Strict Mode (double-invoke do useEffect)
- [ ] **AC9** — `src/lib/types.ts` define `WsMessage` e todos os subtipos de mensagem da Fase 2

- [x] **Task 2** — Criar `src/hooks/useWebSocket.ts` (AC: 1, 2, 3, 4, 5, 6, 8)
  ```typescript
  'use client';
  import { useState, useEffect, useRef, useCallback } from 'react';
  import type { WsIncomingMessage } from '@/lib/types';

  const WS_URL = 'ws://localhost:8888/ws';
  const MAX_RETRIES = 5;
  const BASE_DELAY_MS = 1000;

  export function useWebSocket() {
    const [connected, setConnected] = useState(false);
    const [lastMessage, setLastMessage] = useState<WsIncomingMessage | null>(null);
    const wsRef = useRef<WebSocket | null>(null);
    const retriesRef = useRef(0);
    const timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);
    const unmountedRef = useRef(false);

    const connect = useCallback(() => {
      if (unmountedRef.current) return;

      const ws = new WebSocket(WS_URL);
      wsRef.current = ws;

      ws.onopen = () => {
        if (unmountedRef.current) { ws.close(); return; }
        setConnected(true);
        retriesRef.current = 0;
      };

      ws.onmessage = (e) => {
        try {
          const msg = JSON.parse(e.data) as WsIncomingMessage;
          if (msg.type !== 'ping') setLastMessage(msg);
        } catch { /* ignore malformed */ }
      };

      ws.onclose = () => {
        if (unmountedRef.current) return;
        setConnected(false);
        if (retriesRef.current < MAX_RETRIES) {
          const delay = BASE_DELAY_MS * Math.pow(2, retriesRef.current);
          retriesRef.current++;
          timeoutRef.current = setTimeout(connect, delay);
        }
      };

      ws.onerror = () => ws.close();
    }, []);

    useEffect(() => {
      unmountedRef.current = false;
      connect();
      return () => {
        unmountedRef.current = true;
        if (timeoutRef.current) clearTimeout(timeoutRef.current);
        wsRef.current?.close();
      };
    }, [connect]);

    return { connected, lastMessage };
  }
  ```

## Dev Notes

### React Strict Mode e WebSocket

Em desenvolvimento, React Strict Mode faz double-invoke do useEffect (mount → unmount → mount). O `unmountedRef` garante que a conexão do primeiro mount seja fechada antes da segunda tentativa.

### Backoff exponencial

| Tentativa | Delay |
|-----------|-------|
| 1 | 1s |
| 2 | 2s |
| 3 | 4s |
| 4 | 8s |
| 5 | 16s |

Após 5 falhas, para de tentar. O usuário pode recarregar a página para reiniciar.

### WS URL hardcoded vs env var

Por ser sempre local (port 8888), o URL hardcoded é aceitável para MVP. Fase 5 (npm publish) pode parametrizar via `NEXT_PUBLIC_WS_URL`.

### useCallback + connect

O `connect` é wrapped em `useCallback` com deps `[]` para garantir referência estável entre renders — necessário porque é usado no `useEffect` e no `ws.onclose`.
