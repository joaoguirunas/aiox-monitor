# Story 2.4 — Modo Kanban

## Story

**Como** usuário do aiox-monitor,
**Quero** visualizar todos os agentes ativos organizados em colunas por projeto,
**Para que** eu possa ver de relance quem está fazendo o quê em cada projeto, atualizando em tempo real.

## Acceptance Criteria

- [ ] **AC1** — `/kanban` exibe uma coluna por projeto ativo (projetos com `last_active` nas últimas 24h)
- [ ] **AC2** — Cada coluna mostra o nome do projeto e um contador de agentes ativos
- [ ] **AC3** — Cada agente é um card com: nome, display_name, badge de status, tool atual (se working)
- [ ] **AC4** — Badges de status: `working` (verde pulsante), `idle` (cinza), `break` (amarelo), `offline` (vermelho apagado)
- [ ] **AC5** — Cards de agentes `working` aparecem no topo da coluna
- [ ] **AC6** — Quando `agent:update` chega via WS, o card atualiza status/tool sem re-render de toda a página
- [ ] **AC7** — Quando `event:new` chega, o card do agente relevante "pisca" brevemente (flash de highlight)
- [ ] **AC8** — Estado vazio de coluna: "Nenhum agente ativo" com ícone
- [ ] **AC9** — Estado vazio de página: "Nenhum projeto ativo. Inicie o Claude Code." quando não há projetos
- [ ] **AC10** — Layout responsivo: colunas em scroll horizontal em telas pequenas, grid em telas grandes
- [ ] **AC11** — `npm run typecheck` e `npm run build` passam sem erros

## Tasks / Subtasks

- [ ] **Task 1** — Criar `src/hooks/useKanban.ts` (AC: 1, 6)
  ```typescript
  // Agrega dados de projetos + agentes + WS updates
  export function useKanban() {
    const { projects } = useProjects();
    const { agents } = useAgents(); // sem filtro de projeto
    const { lastMessage } = useWebSocket();
    const [agentState, setAgentState] = useState<Record<number, Agent>>({});

    // Inicializa com dados do fetch
    useEffect(() => {
      const map: Record<number, Agent> = {};
      agents.forEach(a => { map[a.id] = a; });
      setAgentState(map);
    }, [agents]);

    // Atualiza via WS
    useEffect(() => {
      if (lastMessage?.type !== 'agent:update') return;
      const { agent } = lastMessage as WsAgentUpdate;
      setAgentState(prev => ({ ...prev, [agent.id]: agent }));
    }, [lastMessage]);

    // Agrupa agentes por projeto
    const columns = projects.map(project => ({
      project,
      agents: Object.values(agentState)
        .filter(a => a.project_id === project.id)
        .sort((a, b) => {
          if (a.status === 'working' && b.status !== 'working') return -1;
          if (b.status === 'working' && a.status !== 'working') return 1;
          return 0;
        }),
    }));

    return { columns, lastMessage };
  }
  ```

## Dev Notes

### Flash de highlight — UX

O flash indica atividade recente sem ser invasivo. Implementar com:
```typescript
const [flash, setFlash] = useState(false);
useEffect(() => {
  if (!isCurrentAgentEvent) return;
  setFlash(true);
  const t = setTimeout(() => setFlash(false), 600);
  return () => clearTimeout(t);
}, [triggerValue]);
// Classe: flash ? 'ring-2 ring-blue-400' : ''
```

### Agentes "working" primeiro

A ordenação coloca `working` no topo da coluna — o agente mais ativo fica mais visível. Dentro do mesmo status, ordenar por `last_active DESC`.

### Projetos ativos (últimas 24h)

Usar `last_active >= datetime('now', '-24 hours')` — já implementado em `getStats()`. Para o Kanban, filtrar `projects` por `last_active` no frontend para evitar nova query.

```tsx
</div>
```
