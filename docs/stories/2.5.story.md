# Story 2.5 — ProjectSelector

## Story

**Como** usuário do aiox-monitor,
**Quero** selecionar um projeto específico na Navbar e que essa seleção persista entre navegações,
**Para que** o Modo Lista e o Modo Kanban filtrem automaticamente pelo projeto selecionado.

## Acceptance Criteria

- [ ] **AC1** — Navbar exibe um `<select>` de projetos com opção "Todos os projetos" como padrão
- [ ] **AC2** — Lista de projetos no select vem de `GET /api/projects` (somente projetos ativos nas últimas 24h)
- [ ] **AC3** — Projeto selecionado é persistido em `localStorage` com chave `aiox-selected-project`
- [ ] **AC4** — Ao recarregar a página, o select restaura o valor do `localStorage`
- [ ] **AC5** — Quando um projeto é selecionado, o Modo Lista aplica `?project_id=N` automaticamente via `useProjects` context
- [ ] **AC6** — Quando um projeto é selecionado, o Modo Kanban exibe apenas a coluna daquele projeto
- [ ] **AC7** — Quando "Todos os projetos" é selecionado, Kanban e Lista exibem todos
- [ ] **AC8** — O select exibe o `display_name` do projeto (ou `name` se `display_name` for nulo)
- [ ] **AC9** — `npm run typecheck` e `npm run build` passam sem erros

## Tasks / Subtasks

- [ ] **Task 1** — Criar `src/contexts/ProjectContext.tsx` (AC: 3, 4, 5, 6, 7)
  ```typescript
  'use client';
  import { createContext, useContext, useState, useEffect } from 'react';

  interface ProjectContextValue {
    selectedProjectId: number | null;
    setSelectedProjectId: (id: number | null) => void;
  }

  const ProjectContext = createContext<ProjectContextValue>({
    selectedProjectId: null,
    setSelectedProjectId: () => {},
  });

  export function ProjectProvider({ children }: { children: React.ReactNode }) {
    const [selectedProjectId, setSelectedProjectIdState] = useState<number | null>(null);

    // Restore from localStorage on mount
    useEffect(() => {
      const stored = localStorage.getItem('aiox-selected-project');
      if (stored) {
        const id = parseInt(stored, 10);
        if (!isNaN(id)) setSelectedProjectIdState(id);
      }
    }, []);

    const setSelectedProjectId = (id: number | null) => {
      setSelectedProjectIdState(id);
      if (id === null) {
        localStorage.removeItem('aiox-selected-project');
      } else {
        localStorage.setItem('aiox-selected-project', String(id));
      }
    };

    return (
      <ProjectContext.Provider value={{ selectedProjectId, setSelectedProjectId }}>
        {children}
      </ProjectContext.Provider>
    );
  }

  export function useProjectContext() {
    return useContext(ProjectContext);
  }
  ```

  ```typescript
          </option>
        ))}
      </select>
    );
  }
  ```

## Dev Notes

### ProjectContext — Server vs Client

O `ProjectProvider` é um Client Component (`'use client'`). O `localStorage` só existe no browser — o `useEffect` garante que o restore acontece apenas no client-side, evitando hydration mismatch.

### Filtro no Modo Lista — interação com FilterBar

O Modo Lista já tem um FilterBar com dropdowns de projeto. Com o `ProjectSelector` na Navbar, há dois pontos de filtro. Estratégia recomendada:

1. O select da Navbar define o `selectedProjectId` global
2. O FilterBar pode ainda exibir o filtro de projeto, mas deve inicializar com o valor do context
3. Se o usuário muda no FilterBar, atualizar o context também (via `setSelectedProjectId`)

Alternativamente, remover o filtro de projeto do FilterBar se o Navbar selector cobrir o caso de uso. A decisão fica a critério do dev (@dev) ao implementar.

### Projetos ativos

Usar `useProjects()` existente — já retorna projetos ordenados por `last_active DESC`. Não precisamos de nova query.

### Display name

`p.display_name ?? p.name` — `display_name` pode ser null no schema, usar `name` como fallback.
