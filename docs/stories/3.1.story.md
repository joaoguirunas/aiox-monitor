# Story 3.1 — Integração Phaser.js no Next.js

## Story

**Como** aiox-monitor,
**Quero** integrar o Phaser.js no Next.js com dynamic import e SSR desabilitado,
**Para que** o Modo Empresa possa renderizar uma cena isométrica sem quebrar o SSR do Next.js.

## Acceptance Criteria

- [ ] **AC1** — `src/components/empresa/PhaserGame.tsx` exporta um componente React que encapsula uma instância Phaser 3
- [ ] **AC2** — O componente é carregado via `next/dynamic` com `ssr: false` em `src/app/empresa/page.tsx`
- [ ] **AC3** — A cena Phaser renderiza num `<div>` com id `phaser-container` e ocupa 100% do espaço disponível (abaixo da Navbar)
- [ ] **AC4** — A instância Phaser é destruída corretamente no cleanup do `useEffect` (sem memory leaks)
- [ ] **AC5** — A cena inicial é uma `BootScene` que exibe fundo escuro (#1a1a2e) e texto "Carregando escritório..." centralizado
- [ ] **AC6** — `Phaser.AUTO` é usado como renderer (WebGL com fallback Canvas)
- [ ] **AC7** — A configuração Phaser usa `isometric: true` no plugin de tilemap ou projeção manual
- [ ] **AC8** — Resize automático: cena redimensiona ao mudar tamanho da janela (`resize` event + `scale.resize`)
- [ ] **AC9** — `npm run typecheck` e `npm run build` passam sem erros
- [ ] **AC10** — Navegação para `/empresa` mostra a cena Phaser sem erros no console

- [x] **Task 3** — Criar `src/components/empresa/PhaserGame.tsx` (AC: 1, 3, 4)
  ```typescript
  'use client';
  import { useEffect, useRef } from 'react';
  import Phaser from 'phaser';
  import { createGameConfig } from '@/game/config';

  export function PhaserGame() {
    const gameRef = useRef<Phaser.Game | null>(null);

    useEffect(() => {
      if (gameRef.current) return;
      const config = createGameConfig('phaser-container');
      gameRef.current = new Phaser.Game(config);

      return () => {
        gameRef.current?.destroy(true);
        gameRef.current = null;
      };
    }, []);

    return (
      <div
        id="phaser-container"
        className="w-full"
        style={{ height: 'calc(100vh - 56px)' }}
      />
    );
  }
  ```

## Dev Notes

### Phaser.js já está instalado

`phaser: ^3.87.0` já consta em `package.json` como dependência. Não é necessário instalar.

### SSR e Phaser

Phaser acessa `window`, `document` e `WebGL` — incompatível com SSR. O `next/dynamic` com `ssr: false` garante que o componente só carrega no browser. Nunca importar Phaser directamente em componentes que não sejam dinamicamente carregados.

### Projeção Isométrica

Phaser 3 tem suporte nativo a tilemaps isométricos via `Phaser.Tilemaps.Orientation.ISOMETRIC`. Na Story 3.2, o tilemap usará este modo. Nesta story, apenas garantir que a config está preparada (`pixelArt: true` para sprites nítidos).

### Destroy do Game

O `gameRef.current?.destroy(true)` com `true` remove também o canvas do DOM. Essencial para evitar memory leaks no React quando o componente é desmontado (navegação entre páginas).

### Navbar height

A Navbar tem `h-14` (56px). O container Phaser usa `calc(100vh - 56px)` para ocupar o espaço restante. Se a Navbar mudar de altura, ajustar aqui.

### Scale.RESIZE

`Phaser.Scale.RESIZE` faz o canvas redimensionar automaticamente quando o container muda de tamanho. Combinado com `autoCenter`, o jogo fica sempre centrado.
