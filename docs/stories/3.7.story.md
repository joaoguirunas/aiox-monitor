# Story 3.7 — Agente Caminha até Mesa via WebSocket

## Story

**Como** aiox-monitor,
**Quero** que quando um agente receber um evento via WebSocket, o personagem caminhe animado até a mesa correspondente e comece a digitar,
**Para que** o Modo Empresa mostre em tempo real os agentes a trabalhar, com transições visuais fluidas entre estados.

## Acceptance Criteria

- [x] **AC1** — `PhaserGame.tsx` consome `useWebSocket()` e envia eventos `agent:update` para a cena Phaser via bridge
- [x] **AC2** — Quando `agent:update` com `status = 'working'` chega, o agente caminha (walk animation) até uma mesa livre e senta a digitar
- [x] **AC3** — Quando `agent:update` com `status = 'idle'` chega, o agente levanta da mesa, caminha até o lounge, e fica em pé (idle animation)
- [x] **AC4** — Quando `agent:update` com `status = 'break'` chega, o agente vai até a zona do café no lounge
- [x] **AC5** — Quando `agent:update` com `status = 'offline'` chega, o agente caminha até a porta e é removido da cena
- [x] **AC6** — Agentes novos (não existiam na cena) aparecem na porta e caminham até a posição correta
- [x] **AC7** — Transições simultâneas: múltiplos agentes podem caminhar ao mesmo tempo sem conflitos
- [x] **AC8** — A tela da mesa acende com a cor do agente quando ele senta, e apaga quando sai
- [x] **AC9** — O `ProjectSelector` na Navbar filtra os agentes visíveis no Modo Empresa (se projeto selecionado, mostrar apenas agentes desse projeto)
- [x] **AC10** — `npm run typecheck` e `npm run build` passam sem erros

  // Refatorar updateAgentPosition para usar walkTo com async/await
  private async updateAgentPosition(name: string, newStatus: AgentStatus): Promise<void> {
    const sprite = this.sprites.get(name);
    if (!sprite) return;

    if (newStatus === 'offline') {
      // Caminhar até porta, depois remover
      await sprite.walkTo(ENTRANCE_POSITION.tileX, ENTRANCE_POSITION.tileY);
      this.removeAgent(name);
      return;
    }

    if (newStatus === 'working') {
      const deskIdx = this.assignDesk(name);
      if (deskIdx >= 0 && deskIdx < DESK_POSITIONS.length) {
        const desk = DESK_POSITIONS[deskIdx];
        await sprite.walkTo(desk.tileX, desk.tileY);
        sprite.sitDown();
        this.scene.time.delayedCall(400, () => sprite.startTyping());
        if (deskIdx < this.desks.length) {
          this.desks[deskIdx].setScreenOn(getAgentColor(name));
        }
      }
    } else if (newStatus === 'idle') {
      this.releaseDesk(name);
      const pos = this.getLoungePosition(name);
      await sprite.walkTo(pos.tileX, pos.tileY);
      sprite.standIdle();
    } else if (newStatus === 'break') {
      this.releaseDesk(name);
      // Mesa de café é LOUNGE_POSITIONS[2]
      const coffeePos = LOUNGE_POSITIONS[2];
      await sprite.walkTo(coffeePos.tileX, coffeePos.tileY);
      sprite.standIdle();
    }
  }

  // Determina posição no lounge para um agente idle
  private getLoungePosition(name: string): TilePosition {
    const sofaPositions = [LOUNGE_POSITIONS[0], LOUNGE_POSITIONS[1], LOUNGE_POSITIONS[3]];
    const idleAgents = [...this.sprites.entries()]
      .filter(([n, _]) => n !== name)
      .length;
    return sofaPositions[idleAgents % sofaPositions.length];
  }

- [x] **Task 2** — Atualizar `PhaserGame.tsx` — consumir WebSocket e push via bridge (AC: 1, 9)
  ```typescript
  'use client';
  import dynamic from 'next/dynamic';
  import { useEffect, useRef } from 'react';
  import { useWebSocket } from '@/hooks/useWebSocket';
  import { useProjectContext } from '@/contexts/ProjectContext';
  import {
    setGameInstance, clearGameInstance, syncAgents, updateAgent
  } from '@/game/bridge/react-phaser-bridge';
  import type { WsAgentUpdate } from '@/lib/types';

  function PhaserGameInner() {
    const gameRef = useRef<Phaser.Game | null>(null);
    const { lastMessage } = useWebSocket();
    const { selectedProjectId } = useProjectContext();

    // Init game
    useEffect(() => {
      const config = createGameConfig('phaser-container');
      const game = new Phaser.Game(config);
      gameRef.current = game;
      setGameInstance(game);

      // Fetch inicial
      game.events.on('ready', async () => {
        try {
          const url = selectedProjectId
            ? `/api/agents?project_id=${selectedProjectId}`
            : '/api/agents';
          const res = await fetch(url);
          const agents = await res.json();
          const waitForScene = setInterval(() => {
            const scene = game.scene.getScene('OfficeScene');
            if (scene && scene.scene.isActive()) {
              clearInterval(waitForScene);
              syncAgents(agents);
            }
          }, 100);
        } catch { /* silent */ }
      });

      return () => {
        clearGameInstance();
        game.destroy(true);
        gameRef.current = null;
      };
    }, []);

    // Push WS agent:update para Phaser
    useEffect(() => {
      if (!lastMessage || lastMessage.type !== 'agent:update') return;
      const { agent, projectId } = lastMessage as WsAgentUpdate;

      // Filtro de projeto (se selecionado)
      if (selectedProjectId && projectId !== selectedProjectId) return;

      updateAgent(agent);
    }, [lastMessage, selectedProjectId]);

    return (
      <div id="phaser-container" className="w-full" style={{ height: 'calc(100vh - 56px)' }} />
    );
  }

  // Export com dynamic (ssr: false) permanece no page.tsx
  export { PhaserGameInner as PhaserGame };
  ```

## Dev Notes

### Fluxo completo

```
Hook Python → POST /api/events → event-processor.ts
    → agent-tracker.ts → status 'working' + broadcast WS { type: 'agent:update' }
        → useWebSocket (PhaserGame.tsx) → lastMessage
            → bridge.updateAgent(agent)
                → OfficeScene.events.emit('update:agent', agent)
                    → AgentManager.updateAgent(agent)
                        → walkTo(desk) → sitDown() → startTyping()
```

### Transições simultâneas

Cada `walkTo()` é independente (cada sprite tem seu próprio tween). Phaser gere múltiplos tweens em paralelo nativamente. Não há conflito — cada agente caminha no seu próprio ritmo.

### Filtro de projeto

O `selectedProjectId` do `ProjectContext` filtra os events `agent:update` — se um projeto está selecionado, updates de outros projetos são ignorados no Phaser. O fetch inicial também filtra via query param `?project_id=N`.

### Walk queueing

Se um `agent:update` chegar enquanto o agente está a caminhar (tween activo), o `walkTo` para o tween actual e inicia um novo. Não há fila de animações — o destino mais recente ganha. Isto é correcto: o status mais recente é sempre o que importa.

### Idle detector integration

O idle detector (Story 2.6) envia `agent:update` a cada transição de status. O Modo Empresa reage automaticamente:
- working→idle: agente levanta e vai para lounge
- idle→break: agente vai para perto do café
- break→offline: agente vai para porta e desaparece

### Agent que fica offline mas volta

Se um agente fica `offline` (sprite removido), e depois envia um novo evento (volta a `working`), o `AgentManager.updateAgent()` detecta que não existe sprite e chama `createAgent()` — o agente reaparece na porta e caminha até a mesa.
