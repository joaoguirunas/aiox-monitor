# Story 4.1 — Sprites Pixel Art por Agente

## Story

**Como** aiox-monitor,
**Quero** que cada agente tenha um sprite pixel art único com identidade visual distinta (cor, acessório, silhueta),
**Para que** os utilizadores identifiquem visualmente cada agente no escritório sem precisar ler o nome.

## Acceptance Criteria

- [ ] **AC1** — `src/game/objects/AgentSprite.ts` usa spritesheets pixel art (32×32 por frame) em vez de desenho programático via Graphics
- [ ] **AC2** — Existe um spritesheet (PNG + JSON atlas) por agente em `public/sprites/agents/` com frames para: idle (2), walk-down (4), walk-up (4), walk-side (4), sit (2), type (2)
- [ ] **AC3** — Cada agente tem cor de roupa única (mantém mapeamento de `agent-visuals.ts`) e acessório visual distinto (óculos, chapéu, headset, etc.)
- [ ] **AC4** — `BootScene.ts` carrega todos os spritesheets no `preload()` — fallback para sprite programático se asset não carregado
- [ ] **AC5** — `src/game/data/agent-sprite-config.ts` define a configuração visual por agente: spritesheet key, acessório, cor primária, cor secundária
- [ ] **AC6** — O sprite pixel art mantém o status badge (círculo colorido) e nome flutuante existentes
- [ ] **AC7** — O `SpriteGenerator` em `src/game/utils/sprite-generator.ts` gera spritesheets pixel art proceduralmente via Canvas API offline (node script ou runtime fallback)
- [ ] **AC8** — Agentes sem spritesheet customizado usam o sprite base com cor do agente aplicada via tint
- [ ] **AC9** — `npm run typecheck` e `npm run build` passam sem erros
- [ ] **AC10** — Os 11 agentes são visualmente distintos no escritório

- [x] **Task 2** — Criar `src/game/utils/sprite-generator.ts` — gerador procedural de spritesheets pixel art (AC: 7, 8)
  ```typescript
  import { AGENT_SPRITE_CONFIGS, type AgentSpriteConfig } from '../data/agent-sprite-config';

  // Gera um spritesheet pixel art 32x32 para um agente
  // Frames: idle(2) + walk-down(4) + walk-up(4) + walk-side(4) + sit(2) + type(2) = 18 frames
  // Atlas: 6 colunas x 3 linhas = 18 frames em 192x96 px
  export const FRAME_SIZE = 32;
  export const ATLAS_COLS = 6;
  export const ATLAS_ROWS = 3;

  export function generateAgentSpritesheet(
    config: AgentSpriteConfig,
  ): HTMLCanvasElement {
    const canvas = document.createElement('canvas');
    canvas.width = FRAME_SIZE * ATLAS_COLS;
    canvas.height = FRAME_SIZE * ATLAS_ROWS;
    const ctx = canvas.getContext('2d')!;

    // Desabilitar anti-aliasing para estilo pixel art
    ctx.imageSmoothingEnabled = false;

    const framePositions = getFramePositions();

    for (let i = 0; i < framePositions.length; i++) {
      const { col, row, animType, frameIndex } = framePositions[i];
      const offsetX = col * FRAME_SIZE;
      const offsetY = row * FRAME_SIZE;
      drawAgentFrame(ctx, offsetX, offsetY, config, animType, frameIndex);
    }

    return canvas;
  }

  type AnimType = 'idle' | 'walk-down' | 'walk-up' | 'walk-side' | 'sit' | 'type';

  interface FramePosition {
    col: number;
    row: number;
    animType: AnimType;
    frameIndex: number;
  }

  function getFramePositions(): FramePosition[] {
    return [
      // Row 0: idle(2) + walk-down(4)
      { col: 0, row: 0, animType: 'idle', frameIndex: 0 },
      { col: 1, row: 0, animType: 'idle', frameIndex: 1 },
      { col: 2, row: 0, animType: 'walk-down', frameIndex: 0 },
      { col: 3, row: 0, animType: 'walk-down', frameIndex: 1 },
      { col: 4, row: 0, animType: 'walk-down', frameIndex: 2 },
      { col: 5, row: 0, animType: 'walk-down', frameIndex: 3 },
      // Row 1: walk-up(4) + walk-side(2 of 4)
      { col: 0, row: 1, animType: 'walk-up', frameIndex: 0 },
      { col: 1, row: 1, animType: 'walk-up', frameIndex: 1 },
      { col: 2, row: 1, animType: 'walk-up', frameIndex: 2 },
      { col: 3, row: 1, animType: 'walk-up', frameIndex: 3 },
      { col: 4, row: 1, animType: 'walk-side', frameIndex: 0 },
      { col: 5, row: 1, animType: 'walk-side', frameIndex: 1 },
      // Row 2: walk-side(2 of 4) + sit(2) + type(2)
      { col: 0, row: 2, animType: 'walk-side', frameIndex: 2 },
      { col: 1, row: 2, animType: 'walk-side', frameIndex: 3 },
      { col: 2, row: 2, animType: 'sit', frameIndex: 0 },
      { col: 3, row: 2, animType: 'sit', frameIndex: 1 },
      { col: 4, row: 2, animType: 'type', frameIndex: 0 },
      { col: 5, row: 2, animType: 'type', frameIndex: 1 },
    ];
  }

  function drawAgentFrame(
    ctx: CanvasRenderingContext2D,
    offsetX: number, offsetY: number,
    config: AgentSpriteConfig,
    animType: AnimType,
    frameIndex: number,
  ): void {
    // Pixel art character: 32x32 grid
    // Head: 8x8 centered at top
    // Body: 10x12 centered
    // Legs: 2x4 each, offset by frame
    const cx = offsetX + 16; // center X
    const baseY = offsetY + 6; // top of head

    // Skin tone for head
    ctx.fillStyle = hexToRgb(config.skinTone);

    // Head (8x8 circle-ish shape)
    fillPixelCircle(ctx, cx, baseY + 4, 4, config.skinTone);

    // Hair
    drawHair(ctx, cx, baseY, config.hairStyle, config.hairColor);

    // Body (shirt with agent color)
    const bodyY = baseY + 9;
    const bodyOffset = animType === 'sit' ? 2 : 0;
    ctx.fillStyle = hexToRgb(config.primaryColor);
    ctx.fillRect(cx - 5, bodyY + bodyOffset, 10, 10);

    // Arms variation per frame
    const armOffset = (animType === 'type' && frameIndex % 2 === 1) ? 1 : 0;
    ctx.fillStyle = hexToRgb(config.primaryColor);
    ctx.fillRect(cx - 7, bodyY + 1 + bodyOffset + armOffset, 2, 6);
    ctx.fillRect(cx + 5, bodyY + 1 + bodyOffset - armOffset, 2, 6);

    // Legs (vary position for walk frames)
    const legY = bodyY + 10 + bodyOffset;
    const legSpread = getWalkLegSpread(animType, frameIndex);
    ctx.fillStyle = hexToRgb(0x333344);
    ctx.fillRect(cx - 3 - legSpread, legY, 3, animType === 'sit' ? 3 : 5);
    ctx.fillRect(cx + legSpread, legY, 3, animType === 'sit' ? 3 : 5);

    // Accessory
    drawAccessory(ctx, cx, baseY, config.accessory, config.secondaryColor);
  }

  // Helper functions: fillPixelCircle, drawHair, drawAccessory, getWalkLegSpread, hexToRgb
  // (implementação pixel-a-pixel para cada variante)
  ```

## Dev Notes

### Geração procedural vs assets estáticos

Opção escolhida: **geração procedural via Canvas API**. Razões:
1. Não requer artista pixel art
2. Cada agente é automaticamente único (baseado em config)
3. Cores sincronizadas com `agent-visuals.ts`
4. Fácil de ajustar e iterar
5. Assets podem ser exportados como PNG para uso futuro

A desvantagem é qualidade visual — sprites procedurais não são tão bonitos como arte manual. Mas para MVP é suficiente e permite evoluir depois.

### Spritesheet layout

```
Row 0: [idle-0] [idle-1] [walk-down-0] [walk-down-1] [walk-down-2] [walk-down-3]
Row 1: [walk-up-0] [walk-up-1] [walk-up-2] [walk-up-3] [walk-side-0] [walk-side-1]
Row 2: [walk-side-2] [walk-side-3] [sit-0] [sit-1] [type-0] [type-1]
```

18 frames × 32×32 = spritesheet 192×96 pixels.

### Fallback

Se a textura do agente não existir (agente desconhecido), usa `agent-default` com tint aplicado da cor do agente. Isto garante que novos agentes adicionados ao sistema funcionam imediatamente.

### Migração do Graphics para Sprite

A classe `AgentSprite` muda de `Phaser.GameObjects.Graphics` para `Phaser.GameObjects.Sprite`. O container mantém-se (`Phaser.GameObjects.Container`), mas o filho principal passa a ser um Sprite em vez de Graphics.
