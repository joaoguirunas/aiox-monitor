# Story 4.2 — Animações Completas de Transição entre Zonas

## Story

**Como** aiox-monitor,
**Quero** que os agentes usem animações frame-based completas ao caminhar entre zonas (trabalho, lounge, entrada), com direção correcta e efeitos visuais de transição,
**Para que** o Modo Empresa tenha movimento fluido e natural, tornando o escritório virtual imersivo.

## Acceptance Criteria

- [ ] **AC1** — O sprite muda de animação conforme a direcção do movimento: walk-down, walk-up, walk-side (espelhado para esquerda/direita)
- [ ] **AC2** — Ao chegar ao destino, o sprite transiciona suavemente para a animação final (idle, sit, type) sem salto visual
- [ ] **AC3** — O walkTo calcula a direcção dominante (deltaX vs deltaY) para escolher a animação correcta
- [ ] **AC4** — Walk-side é espelhado horizontalmente (`flipX`) quando o agente caminha para a esquerda
- [ ] **AC5** — Efeito visual ao cruzar fronteira de zona: breve flash sutil (alpha pulse) no sprite
- [ ] **AC6** — A velocidade do walk é proporcional à distância (mínimo 500ms, máximo 3000ms) com easing natural
- [ ] **AC7** — Múltiplos agentes a caminhar simultaneamente mantêm animações independentes e fluidas
- [ ] **AC8** — O AgentManager gere fila de transições: se novo status chega durante walk, cancela walk actual e inicia novo
- [ ] **AC9** — `npm run typecheck` e `npm run build` passam sem erros
- [ ] **AC10** — Teste visual: enviar 5 eventos curl simultâneos → 5 agentes caminham com animações correctas

  ```typescript
  walkTo(tileX: number, tileY: number): Promise<void> {
    return new Promise((resolve) => {
      this.stopCurrentAnimation();
      this.animState = 'walk';

      const { x: targetX, y: targetY } = tileToPixel(tileX, tileY);
      const deltaX = targetX - this.x;
      const deltaY = targetY - this.y;
      const distance = Phaser.Math.Distance.Between(this.x, this.y, targetX, targetY);
      const duration = Phaser.Math.Clamp(distance * 4, 500, 3000);

      // Determinar direcção dominante
      const direction = this.getWalkDirection(deltaX, deltaY);
      this.playWalkAnimation(direction);

      // Flip horizontal para walk-side esquerda
      if (direction === 'side' && deltaX < 0) {
        this.sprite.setFlipX(true);
      } else {
        this.sprite.setFlipX(false);
      }

      this.currentTween = this.scene.tweens.add({
        targets: this,
        x: targetX,
        y: targetY,
        duration,
        ease: 'Power1',
        onUpdate: () => {
          this.setDepth(this.y + 1);
          this.checkZoneCrossing();
        },
        onComplete: () => {
          this.currentTween = null;
          this.sprite.setFlipX(false);
          this.stopWalkAnimation();
          this.animState = 'idle';
          resolve();
        },
      });
    });
  }

  private getWalkDirection(deltaX: number, deltaY: number): 'down' | 'up' | 'side' {
    const absDx = Math.abs(deltaX);
    const absDy = Math.abs(deltaY);

    // Em isométrico, "down" = deltaY > 0, "up" = deltaY < 0
    // "side" = quando o movimento horizontal é dominante
    if (absDx > absDy * 1.5) return 'side';
    if (deltaY > 0) return 'down';
    return 'up';
  }

  private playWalkAnimation(direction: 'down' | 'up' | 'side'): void {
    const config = getAgentSpriteConfig(this.agentName);
    const animKey = `${config.key}-walk-${direction}`;
    if (this.scene.anims.exists(animKey)) {
      this.sprite.play(animKey);
    }
  }

  private stopWalkAnimation(): void {
    if (this.sprite.anims.isPlaying) {
      this.sprite.anims.stop();
    }
    // Voltar para idle frame
    const config = getAgentSpriteConfig(this.agentName);
    const idleKey = `${config.key}-idle`;
    if (this.scene.anims.exists(idleKey)) {
      this.sprite.play(idleKey);
    }
  }
  ```

  private lastZone: string | null = null;

  private checkZoneCrossing(): void {
    const { tileX, tileY } = pixelToTile(this.x, this.y);
    const currentZone = getZoneForTile(tileX, tileY);
    if (currentZone && currentZone !== this.lastZone && this.lastZone !== null) {
      // Flash sutil ao cruzar zona
      this.scene.tweens.add({
        targets: this.sprite,
        alpha: 0.6,

        yoyo: true,
        ease: 'Sine.easeInOut',
      });
    }
    this.lastZone = currentZone;
  }

- [x] **Task 3** — Criar `pixelToTile` em `iso-utils.ts` (AC: 5)
  ```typescript
  // Inverso de tileToPixel — converte pixel coords para tile coords aproximadas
  export function pixelToTile(px: number, py: number): { tileX: number; tileY: number } {
    // Inverso da fórmula isométrica 2:1
    // px = (tileX - tileY) * (TILE_WIDTH / 2) + offsetX
    // py = (tileX + tileY) * (TILE_HEIGHT / 2) + offsetY
    const adjustedX = px - ISO_OFFSET_X;
    const adjustedY = py - ISO_OFFSET_Y;
    const tileX = Math.round((adjustedX / (TILE_WIDTH / 2) + adjustedY / (TILE_HEIGHT / 2)) / 2);
    const tileY = Math.round((adjustedY / (TILE_HEIGHT / 2) - adjustedX / (TILE_WIDTH / 2)) / 2);
    return { tileX, tileY };
  }
  ```

## Dev Notes

### Direcção do walk em isométrico

Em projeção isométrica 2:1, os eixos estão rotacionados 45°. A heurística para determinar direcção:
- Se `|deltaX| > |deltaY| * 1.5` → walk-side (horizontal dominante)
- Se `deltaY > 0` → walk-down (caminha "para baixo" no ecrã)
- Senão → walk-up

O walk-side é reutilizado para esquerda e direita via `flipX`.

### Zona crossing

O `checkZoneCrossing()` é chamado no `onUpdate` do tween de walk. Usa `pixelToTile()` para converter a posição actual do sprite em coordenadas de tile, e depois `getZoneForTile()` para determinar a zona. Se mudou de zona, aplica um flash sutil (alpha pulse 0.6→1.0, 150ms).

### Cancelamento de walk

Se um novo `agent:update` chega enquanto o agente caminha (tween activo), o `walkTo` do `AgentSprite` já chama `stopCurrentAnimation()` que para o tween actual. A Promise do walk anterior nunca resolve com `onComplete`, mas isso é seguro — o guard `this.sprites.has(name)` no AgentManager protege contra acções em sprites removidos.

### Performance

Cada sprite tem o seu próprio tween e animação independente. Phaser gere até dezenas de tweens simultâneos sem problema. As animações frame-based são mais leves que tweens (apenas mudam o frame index do spritesheet).
