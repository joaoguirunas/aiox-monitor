# Story 5.1 — PM2 — Startup Automático no Boot do Mac

## Story

**Como** utilizador do aiox-monitor,
**Quero** que o servidor inicie automaticamente no boot do Mac via PM2, sem precisar abrir terminal manualmente,
**Para que** a monitorização dos agentes esteja sempre disponível sem intervenção.

## Acceptance Criteria

- [ ] **AC1** — PM2 adicionado como dependency no `package.json`
- [ ] **AC2** — Ficheiro `ecosystem.config.cjs` configurado para o aiox-monitor (nome, script, env, logs)
- [ ] **AC3** — Script `npm run pm2:start` inicia o servidor com PM2
- [ ] **AC4** — Script `npm run pm2:stop` para o servidor PM2
- [ ] **AC5** — Script `npm run pm2:logs` mostra logs em tempo real
- [ ] **AC6** — Script `npm run pm2:startup` configura PM2 para iniciar no boot do macOS (launchd)
- [ ] **AC7** — O servidor PM2 usa `NODE_ENV=production` e a porta 8888
- [ ] **AC8** — PM2 reinicia o servidor automaticamente em caso de crash (max 10 restarts)
- [ ] **AC9** — Logs do PM2 são escritos em `logs/` (stdout + stderr separados)
- [ ] **AC10** — `npm run typecheck` e `npm run build` passam sem erros

## Dev Notes

- PM2 com `tsx` em vez de `node` porque o server.ts é TypeScript
- `ecosystem.config.cjs` (CJS) para compatibilidade com PM2
- `pm2 startup launchd` é específico para macOS — cria o plist em `~/Library/LaunchAgents/`
- Após `pm2:startup`, o utilizador deve executar o comando que o PM2 imprime (com sudo)
