# Story 5.4 — Limpeza Automática de Eventos Antigos

## Story

**Como** aiox-monitor,
**Quero** que eventos com mais de 30 dias sejam apagados automaticamente para evitar crescimento infinito do SQLite,
**Para que** o banco de dados mantenha um tamanho razoável e o desempenho não degrade ao longo do tempo.

## Acceptance Criteria

- [ ] **AC1** — Módulo `src/server/cleanup.ts` com lógica de limpeza de eventos antigos
- [ ] **AC2** — A limpeza remove eventos da tabela `events` com `created_at` > 30 dias
- [ ] **AC3** — A limpeza também remove sessões concluídas com `ended_at` > 30 dias
- [ ] **AC4** — A limpeza roda automaticamente 1x por dia (intervalo configurável)
- [ ] **AC5** — A limpeza é disparada no startup do servidor (com delay de 60s para não impactar boot)
- [ ] **AC6** — O número de registos removidos é logado no console
- [ ] **AC7** — O período de retenção (30 dias) é lido da `company_config` ou usa default
- [ ] **AC8** — A tabela `company_config` ganha coluna `event_retention_days` (default 30)
- [ ] **AC9** — Script manual `npm run cleanup` para forçar limpeza imediata
- [ ] **AC10** — `npm run typecheck` e `npm run build` passam sem erros

- [x] **Task 2** — Criar `src/server/cleanup.ts` (AC: 1, 2, 3, 6)
  ```typescript
  import { db } from '@/lib/db';

  export function cleanupOldEvents(retentionDays: number = 30): { events: number; sessions: number } {
    const cutoff = new Date(Date.now() - retentionDays * 24 * 60 * 60 * 1000).toISOString();

    const eventsResult = db.prepare('DELETE FROM events WHERE created_at < ?').run(cutoff);
    const sessionsResult = db.prepare(
      "DELETE FROM sessions WHERE ended_at IS NOT NULL AND ended_at < ?"
    ).run(cutoff);

    const events = eventsResult.changes;
    const sessions = sessionsResult.changes;

    if (events > 0 || sessions > 0) {
      console.log(`[cleanup] Removed ${events} events and ${sessions} sessions older than ${retentionDays} days`);
    }

    return { events, sessions };
  }
  ```

- [x] **Task 3** — Integrar cleanup no `server.ts` com intervalo diário (AC: 4, 5, 7)
  ```typescript
  import { cleanupOldEvents } from '@/server/cleanup';
  import { getCompanyConfig } from '@/lib/queries';

  // Cleanup inicial com delay de 60s
  setTimeout(() => {
    const config = getCompanyConfig();
    cleanupOldEvents(config.event_retention_days);

    // Repetir a cada 24h
    setInterval(() => {
      const config = getCompanyConfig();
      cleanupOldEvents(config.event_retention_days);
    }, 24 * 60 * 60 * 1000);
  }, 60_000);
  ```

## Dev Notes

- Usar `DELETE` directo é eficiente em SQLite para tabelas sem muitas FK cascades
- Após delete em massa, podemos adicionar `VACUUM` mas só se necessário (bloqueia DB)
- O `retentionDays` default de 30 é suficiente para a maioria dos casos
- A limpeza não afecta agentes nem projectos — apenas eventos e sessões antigas
