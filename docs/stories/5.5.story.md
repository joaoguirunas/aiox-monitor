# Story 5.5 — Responsive Design e Polish Visual

## Story

**Como** utilizador do aiox-monitor,
**Quero** que a interface funcione bem em diferentes tamanhos de ecrã e tenha acabamento visual profissional,
**Para que** o dashboard seja utilizável em monitores grandes, laptops e tablets, com aparência polida.

## Acceptance Criteria

- [x] **AC1** — Navbar responsiva: em mobile, links colapsam num menu hamburger
- [x] **AC2** — Modo Lista: tabela responsiva com scroll horizontal em ecrãs pequenos
- [x] **AC3** — Modo Kanban: colunas empilham verticalmente em ecrãs < 768px
- [x] **AC4** — Modo Empresa: Phaser canvas redimensiona com a janela (`resize` event)
- [x] **AC5** — Config page: formulário stack vertical em mobile, mantém usabilidade
- [x] **AC6** — Favicon e meta tags `<head>` (title, description, og:image)
- [x] **AC7** — Loading states: skeleton/spinner em todas as páginas durante fetch inicial
- [x] **AC8** — Empty states: mensagens informativas quando não há dados (sem projectos, sem eventos)
- [x] **AC9** — Error states: feedback visual quando API falha (banner ou toast)
- [x] **AC10** — `npm run typecheck` e `npm run build` passam sem erros

## Dev Notes

- Tailwind breakpoints: sm (640px), md (768px), lg (1024px), xl (1280px)
- Phaser `scale.resize()` recalcula o canvas — os objectos mantêm posição relativa
- Favicon pode ser um simples ícone pixel art gerado ou SVG inline
- Loading skeletons: usar `animate-pulse` do Tailwind com blocos cinza
