# Story 5.6 — Preparação para npm publish

## Story

**Como** maintainer do aiox-monitor,
**Quero** que o package.json esteja preparado para publicação no npm com bin entry, versão semântica e files corretos,
**Para que** os utilizadores possam instalar com `npx aiox-monitor` ou `npm install -g aiox-monitor`.

## Acceptance Criteria

- [x] **AC1** — `package.json` tem campos completos: name, version, description, author, license, repository, keywords, homepage
- [x] **AC2** — Campo `bin` aponta para um script CLI que inicia o servidor
- [x] **AC3** — Script `bin/aiox-monitor.js` com shebang que executa o server
- [x] **AC4** — Campo `files` lista apenas os ficheiros necessários para distribuição (exclui docs, stories, .claude)
- [x] **AC5** — Versão `1.0.0` definida no package.json
- [x] **AC6** — Campo `engines` especifica `node >= 18`
- [x] **AC7** — `.npmignore` exclui ficheiros desnecessários (docs, stories, .claude, .aiox-core, tests)
- [x] **AC8** — `npm pack` gera um tarball limpo sem ficheiros desnecessários
- [x] **AC9** — O bin script funciona: `node bin/aiox-monitor.js` inicia o servidor
- [x] **AC10** — `npm run typecheck` e `npm run build` passam sem erros

## Dev Notes

- `npx aiox-monitor` será o comando de instalação global mais simples
- O bin script usa `tsx` para executar TypeScript directamente
- `files` whitelist é preferível a `.npmignore` blacklist — mais seguro
- O tarball deve ter < 5MB para npm publish
- Não publicar realmente no npm nesta story — apenas preparar
