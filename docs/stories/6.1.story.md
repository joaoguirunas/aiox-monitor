# Story 6.1 — JSONL Transcript Intelligence

## Story

**Como** utilizador do aiox-monitor,
**Quero** ver em tempo real qual tool cada agente está a executar, ser notificado quando um agente está à espera de permissão, e ver efeitos visuais de spawn/despawn,
**Para que** o dashboard reflita com precisão o estado real de cada sessão Claude Code e eu saiba exactamente quando preciso intervir.

## Context

Inspirado no projecto [pixel-agents](https://github.com/pablodelucca/pixel-agents), que usa JSONL transcript parsing para obter visibilidade granular das sessões Claude Code. Actualmente o aiox-monitor sabe apenas se um terminal está `processing`/`active`/`inactive` via hooks — não sabe qual tool está a correr nem se o agente está bloqueado à espera de aprovação.

O Claude Code escreve transcripts JSONL em `~/.claude/projects/<project-hash>/<session-id>.jsonl` com registos `assistant` (tool_use), `user` (tool_result), e `system` (turn_duration). Ao monitorizar estes ficheiros incrementalmente, conseguimos:
1. Saber exactamente qual ferramenta está em uso (Bash, Read, Write, Grep, etc.)
2. Detectar quando o agente está à espera de permissão (timeout sem actividade JSONL)
3. Detectar quando um turno termina (turn_duration system message)

## Dev Notes

### Referência: pixel-agents transcript parser
O projecto [pixel-agents](https://github.com/pablodelucca/pixel-agents) tem uma implementação de referência em:
- `src/transcriptParser.ts` — Parser de JSONL com extracção de tool_use/tool_result/turn_duration
- `src/fileWatcher.ts` — Triple-redundant file watching (fs.watch + fs.watchFile + polling)
- `src/timerManager.ts` — Permission detection via timeout

### Formato JSONL do Claude Code
```jsonl
{"type":"assistant","message":{"content":[{"type":"tool_use","id":"toolu_xxx","name":"Bash","input":{"command":"npm test"}}]}}
{"type":"user","message":{"content":[{"type":"tool_result","tool_use_id":"toolu_xxx","content":"..."}]}}
{"type":"system","subtype":"turn_duration","duration_ms":1234}
```

### Campos novos no schema
```sql
ALTER TABLE terminals ADD COLUMN current_tool_detail TEXT;
ALTER TABLE terminals ADD COLUMN waiting_permission INTEGER DEFAULT 0;
```

### Mapeamento Tool → Status
| Tool Name | Status Template |
|-----------|----------------|
| Bash | "Running: {command first 50 chars}" |
| Read | "Reading {file basename}" |
| Write | "Writing {file basename}" |
| Edit | "Editing {file basename}" |
| Grep | "Searching code" |
| Glob | "Finding files" |
| WebFetch | "Fetching web page" |
| WebSearch | "Searching the web" |
| Agent/Task | "Spawning sub-agent" |
| Other | "Using {toolName}" |

### Associação JSONL → Terminal
O path JSONL é `~/.claude/projects/<project-hash>/<session-id>.jsonl`. O `session-id` deve ser cruzado com o campo `session_id` da tabela `terminals` (já alimentado pelo hook `on_tool_start`). Se não houver match directo, usar o PID do processo que tem o ficheiro aberto (`lsof`).
