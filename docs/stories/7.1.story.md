# Story 7.1 — Ganga Dashboard

## Story

**Como** operador do aiox-monitor,
**Quero** ver dentro da Config o que o Ganga Ativo está a fazer — logs de auto-respostas, prompts bloqueados, último heartbeat e poder mudar o scope —
**Para que** o sistema autônomo deixe de ser uma caixa preta e eu tenha confiança e controlo total sobre o que é auto-respondido nos meus terminais.

## Context

O Ganga Ativo corre em background a cada 5 minutos (`src/server/ganga/ganga-engine.ts`). Classifica prompts como safe/blocked/ambiguous e auto-responde os seguros via AppleScript. Hoje na Config existe apenas o toggle on/off — **zero visibilidade** sobre o que está a acontecer.

**Backend já implementado:**
- `GET /api/ganga` → retorna `{ enabled, scope, logs: GangaLog[], stats: { autoResponses, blocked, skipped } }` (`src/app/api/ganga/route.ts:1-22`)
- WS broadcast `ganga:heartbeat` com summary (`GangaHeartbeat`) a cada ciclo (`src/server/ganga/ganga-engine.ts`)
- `ganga_scope` persiste em DB via `PUT /api/company-config` (já tem validação: `src/app/api/company-config/route.ts:44-48`)

**O que falta:** Consumer React na Config page que exiba estes dados.

## Acceptance Criteria

- [ ] AC1: **Scope toggle** — Adicionar toggle ou segmented control para `ganga_scope` (safe-only / safe-and-ambiguous) na secção Ganga da Config. Ao guardar, o valor é enviado no PUT body junto com os restantes campos. Localização: logo abaixo do toggle de enabled (`config/page.tsx`, após a `<div>` do heartbeat indicator actual)
- [ ] AC2: **Contadores 24h** — 3 badges inline mostrando: auto-respondidos (verde), bloqueados (vermelho), ignorados (amarelo). Dados vêm de `GET /api/ganga → stats`. Atualizam ao receber WS `ganga:heartbeat` (campo `summary.autoResponses`, `summary.blockedPrompts`, `summary.skippedPrompts`)
- [ ] AC3: **Log table** — Tabela colapsável com últimos 20 logs. Colunas: timestamp (TimeAgo), classificação (badge colorido: safe=verde, blocked=vermelho, ambiguous=amarelo), prompt (truncado 80 chars com tooltip full), acção, terminal_id. Dados de `GET /api/ganga → logs`. **Lazy load**: fetch apenas quando user expande a secção (não no init da página)
- [ ] AC4: **Heartbeat indicator** — Linha "Último heartbeat: Xm atrás" que actualiza em real-time. Fonte: WS mensagem `ganga:heartbeat → summary.lastCheck`. Se > 10 min sem heartbeat, mudar cor para amarelo warning. Se ganga desactivado, mostrar "Desactivado" em cinza
- [ ] AC5: **Dimmed state** — Quando `ganga_enabled === 0`, toda a secção Ganga Dashboard (scope toggle, counters, logs, heartbeat) fica visível mas com opacity 0.4 e pointer-events none, excepto o toggle de enabled
- [ ] AC6: **WS integration** — O `useWebSocket` hook na Config page já escuta mensagens. Adicionar handler para `type: 'ganga:heartbeat'`: actualizar contadores + heartbeat timestamp sem re-fetch HTTP. Tipo WS já definido: `WsGangaHeartbeat` em `src/lib/types.ts:190-193`
