# Story 7.2 — Config UX Polish

## Story

**Como** operador do aiox-monitor,
**Quero** saber visualmente quando tenho mudanças por guardar, ser avisado se tento sair sem guardar, e não ver warnings no build,
**Para que** não perca configurações por distração e o projecto se mantenha limpo de tech debt.

## Context

A Config page (`src/app/empresa/config/page.tsx`) tem 6 build warnings acumulados: 2 unused imports, 2 `<img>` sem next/image, 2 unused imports no `ganga-engine.ts`. Além disso, não há detecção de "dirty state" — o user pode editar e navegar sem guardar, perdendo tudo.

## Acceptance Criteria

### Dirty State

- [ ] AC1: **Dirty detection** — Manter referência do config original (snapshot no load) e comparar com config actual via `JSON.stringify`. Se diferente, marcar `isDirty = true`. Skins também participam no dirty check (comparar skinConfig com snapshot do skinConfig no load)
- [ ] AC2: **Botão visual** — Quando `isDirty`, o botão "Guardar" muda: fundo de `bg-accent-blue` para `bg-amber-500` (ou similar que se destaque), e mostra "Guardar *" ou um dot indicator. Quando não dirty, botão fica com opacity reduzida ou estilo neutro
- [ ] AC3: **Beforeunload guard** — Registar `window.addEventListener('beforeunload', handler)` quando `isDirty === true`. O handler chama `e.preventDefault()` para mostrar aviso nativo do browser. Remover listener quando dirty volta a false (após save bem sucedido) ou no cleanup do useEffect
- [ ] AC4: **Reset on save** — Após save bem sucedido, actualizar snapshot para o novo config (para que dirty volte a false)

### Debounce & Protection

- [ ] AC5: **Save protection** — O botão Guardar já tem `disabled={saving}` — confirmar que se mantém. Adicionalmente, handleSave deve retornar imediatamente se `saving` já é true (guard clause no início da função, antes do setSaving)

### Build Warnings Cleanup

- [ ] AC6: **Remove unused `SkinDefinition` import** — Em `config/page.tsx:9`, o import `type SkinDefinition` não é usado. Remover do import statement
- [ ] AC7: **Remove unused `SKIN_OPTIONS` variable** — Em `config/page.tsx:20-24`, a variável `SKIN_OPTIONS` está definida mas nunca é referenciada. Remover completamente (linhas 20-24)
- [ ] AC8: **Fix `<img>` warnings** — Em `config/page.tsx:316` e `config/page.tsx:323`, substituir `<img>` por `next/image` (`Image` component), ou adicionar `/* eslint-disable-next-line @next/next/no-img-element */` com comentário justificativo (pixel art sprites precisam de `image-rendering: pixelated` que funciona melhor com `<img>`)
- [ ] AC9: **Remove unused imports em ganga-engine.ts** — Em `src/server/ganga/ganga-engine.ts:16`, os tipos `Agent` e `Project` são importados mas não usados. Remover do import statement
- [ ] AC10: **Verificação** — `npm run build` completa com 0 warnings nos ficheiros `src/app/empresa/config/page.tsx` e `src/server/ganga/ganga-engine.ts`

### Implementação do dirty state (guidelines)

```typescript
// Novo state
const [configSnapshot, setConfigSnapshot] = useState<string>('');
const [skinSnapshot, setSkinSnapshot] = useState<string>('');

// No useEffect de load:
setConfigSnapshot(JSON.stringify(loadedConfig));
setSkinSnapshot(JSON.stringify(loadedSkins));

// Derived:
const isDirty = JSON.stringify(config) !== configSnapshot
             || JSON.stringify(skinConfig) !== skinSnapshot;

// Beforeunload:
useEffect(() => {
  if (!isDirty) return;
  const handler = (e: BeforeUnloadEvent) => { e.preventDefault(); };
  window.addEventListener('beforeunload', handler);
  return () => window.removeEventListener('beforeunload', handler);
}, [isDirty]);

// Após save sucesso:
setConfigSnapshot(JSON.stringify(config));
setSkinSnapshot(JSON.stringify(skinConfig));
```

### Decisão sobre `<img>` vs `next/image`

Os sprites de skins usam `image-rendering: pixelated` e são PNGs de 96px. O `next/image` adiciona optimização que pode degradar pixel art. **Recomendação:** usar eslint-disable com justificação em comentário (pixel art sprites requerem rendering pixelated sem optimização de imagem).
