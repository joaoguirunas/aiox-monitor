# Story 7.4 — Event Retention Config

## Story

**Como** operador do aiox-monitor,
**Quero** configurar quantos dias de eventos são mantidos e poder forçar uma limpeza manual,
**Para que** controle o tamanho da base de dados e liberte espaço quando necessário.

## Context

A coluna `event_retention_days` já existe na tabela `company_config` com default 30 dias (`src/lib/schema.ts:84`). O cleanup automático já corre a cada 24h no servidor (`server.ts:79-91`), usando `cleanupOldEvents(retentionDays)` de `src/server/cleanup.ts`. O que falta é:

1. UI na Config page para ajustar o valor
2. API endpoint (ou extensão do existente) para triggar cleanup manual
3. Feedback visual com contagem de eventos actuais

## Acceptance Criteria

### UI

- [ ] AC1: **Slider de retention** — Adicionar `RangeField` (componente já existe na Config page) para `event_retention_days`. Range: 7-365 dias, step 1. Display: `"{N} dias"`. Localização: na secção Settings, após os timeouts de inactividade
- [ ] AC2: **Info de eventos** — Abaixo do slider, mostrar em texto pequeno: "X eventos armazenados" (total count). Dados vêm de um novo campo no `GET /api/stats` ou num endpoint dedicado. Se 0 eventos, mostrar "Nenhum evento armazenado"
- [ ] AC3: **Botão cleanup manual** — Botão "Limpar eventos antigos" ao lado do info. Estilo: `border-amber-500/30 text-amber-400` (mesmo padrão do botão "Limpar" dos projetos). Ao clicar, chama endpoint de cleanup. Mostra toast com resultado: "X eventos e Y sessões removidos"

### API

- [ ] AC4: **API validation** — Em `PUT /api/company-config`, validar `event_retention_days`: deve ser número entre 7 e 365. Retornar 400 se fora dos bounds. Adicionar ao bloco de validação existente (pattern dos outros campos em `route.ts:24-35`)
- [ ] AC5: **Cleanup endpoint** — Criar `POST /api/cleanup` (novo route file: `src/app/api/cleanup/route.ts`). Lê `event_retention_days` de `getCompanyConfig()`, chama `cleanupOldEvents(days)`, retorna `{ success: true, events: N, sessions: N }`. Proteger com verificação que o campo é válido

### Backend

- [ ] AC6: **Event count query** — Adicionar função `getEventCount()` em `queries.ts`: `SELECT COUNT(*) as count FROM events`. Expor no `GET /api/stats` como novo campo `totalEvents` (já retorna `eventsToday`, adicionar `totalEvents` ao mesmo response)
