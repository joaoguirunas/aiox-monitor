# Story 7.5 — Logo & Branding

## Story

**Como** operador do aiox-monitor,
**Quero** fazer upload do logo da minha empresa na Config e vê-lo no escritório isométrico,
**Para que** o dashboard tenha a identidade visual da minha organização.

## Context

A coluna `logo_path TEXT` existe na tabela `company_config` desde a criação (`src/lib/schema.ts:78`), mas nunca foi implementada — é dead code. O `CompanyConfig` type já inclui `logo_path: string | null` (`src/lib/types.ts:80`).

**Decisão de storage:** Para um sistema local single-user, base64 no DB é mais simples que gestão de ficheiros (sem preocupação com paths, permissões, ou cleanup). Máx 200KB em base64 = ~270KB de texto — aceitável para SQLite.

**Alternativa:** Se esta story for considerada desnecessária, a alternativa é remover `logo_path` do schema e tipos (cleanup de dead code). Esta decisão deve ser tomada antes da implementação.

## Acceptance Criteria

### UI — Config Page

- [ ] AC1: **File input** — Secção "Logo da Empresa" na Config com input de ficheiro. Aceita PNG, JPG, SVG. Máx 200KB. Mostrar tamanho máximo na label. Estilo: mesma estética dos outros SettingBlocks
- [ ] AC2: **Preview** — Ao seleccionar ficheiro, mostrar preview inline (64x64px, object-contain). Se já tem logo guardado, mostrar o logo actual ao carregar a página
- [ ] AC3: **Remove button** — Se tem logo, mostrar botão "Remover" que limpa o campo (set `logo_path = null`). Confirmar com toast
- [ ] AC4: **Validation client-side** — Verificar tipo (accept=".png,.jpg,.jpeg,.svg") e tamanho (200KB) antes de converter. Mostrar toast de erro se inválido

### API

- [ ] AC5: **PUT /api/company-config** — Aceitar campo `logo_path` no body. Valor: string base64 com data URI prefix (`data:image/png;base64,...`) ou `null` para remover. Validar tamanho máximo: 300KB (base64 é ~33% maior que binário)
- [ ] AC6: **GET /api/company-config** — Já retorna `logo_path` — confirmar que o base64 string completo vem no response

### Game — Escritório Isométrico

- [ ] AC7: **Logo no escritório** — Exibir o logo como textura Phaser no escritório. Posição sugerida: na parede da entrada (perto da porta) ou como placa suspensa. Usar `Phaser.Textures.addBase64()` para carregar do base64. Tamanho display: 48x48px. Se não tem logo, não mostrar nada (não mostrar placeholder)
- [ ] AC8: **Reactivity** — Quando o logo muda (via `PUT` → broadcast), o Phaser deve actualizar sem reload da página. Usar bridge event `set:logo`

### Conversão client-side (guidelines)

```typescript
function fileToBase64(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => resolve(reader.result as string);
    reader.onerror = reject;
    reader.readAsDataURL(file);
  });
}

// No handler do input:
const file = e.target.files?.[0];
if (!file) return;
if (file.size > 200 * 1024) { showToast('error', 'Logo max 200KB'); return; }
const base64 = await fileToBase64(file);
setConfig({ ...config, logo_path: base64 });
```
