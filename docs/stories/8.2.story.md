# Story 8.2 — Eliminate First-Project Fallback in syncSystemTerminals

## Story

**Como** sistema de monitorização,
**Quero** que `syncSystemTerminals` ignore terminais detectados sem match de projecto verificável,
**Para que** não sejam criados terminais fantasma atribuídos a projectos errados.

## Context

**Bug actual:** `terminal-tracker.ts:175` — quando o system detection encontra um processo Claude mas o window title não contém o nome de nenhum projecto, o fallback `projects[0].id` atribui o terminal ao primeiro projecto da lista. Isto cria terminais fantasma visíveis na UI e na Empresa.

**Referência:** `docs/Terminais.md` secção 5.4 | Epic 8 Bug Ref 5.4

## Acceptance Criteria

- [x] **AC1** — Em `syncSystemTerminals()` (`src/server/terminal-tracker.ts`), substituir `projectId = matched?.id ?? projects[0].id` por `projectId = matched?.id` seguido de `if (!projectId) continue`
- [x] **AC2** — Terminais detectados pelo sistema sem match de projecto por window title são silenciosamente ignorados (não criados na DB)
- [x] **AC3** — Terminais com match de projecto por window title continuam a ser criados normalmente
- [x] **AC4** — `npm run build` passa sem erros

## Dev Notes

### Código actual (BUGADO)
```typescript
// terminal-tracker.ts:175
projectId = matched?.id ?? projects[0].id;
```

### Correcção
```typescript
projectId = matched?.id;
if (!projectId) continue; // Don't guess — skip unknown terminals
```

**Impacto:** Uma linha mudada + uma adicionada. Zero risco de regressão — apenas previne criação de terminais sem projecto verificado.
