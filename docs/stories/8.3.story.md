# Story 8.3 — Improve JSONL-to-Terminal Matching Accuracy

## Story

**Como** sistema de monitorização,
**Quero** que o JSONL watcher associe tool details ao terminal que está realmente a executar a tool,
**Para que** com múltiplos terminais no mesmo projecto, cada card mostre os dados da sua própria sessão.

## Context

**Bug actual:** Quando o match por `session_id` falha, `findTerminalByProjectDir` selecciona o terminal mais recente (`ORDER BY last_active DESC LIMIT 1`). Com múltiplos terminais no mesmo projecto, tool details vão sempre para o terminal com `last_active` mais recente — não para o que está efectivamente a executar.

**Pré-requisito:** Story 8.1 (fix path conversion) deve estar completa para que `findTerminalByProjectDir` encontre o projecto correcto.

**Referência:** `docs/Terminais.md` secção 5.3 | Epic 8 Bug Ref 5.3

## Dev Notes

### Abordagem para AC1
A query actual:
```sql
SELECT id, project_id FROM terminals WHERE project_id = ? AND status != 'inactive' ORDER BY last_active DESC LIMIT 1
```

Deve ser substituída por duas queries sequenciais:
```sql
-- 1. Prefer processing terminal
SELECT id, project_id FROM terminals WHERE project_id = ? AND status = 'processing' ORDER BY last_active DESC LIMIT 1
-- 2. Fallback to any non-inactive
SELECT id, project_id FROM terminals WHERE project_id = ? AND status != 'inactive' ORDER BY last_active DESC LIMIT 1
```

### Limitação conhecida
Com 2+ terminais em `processing` no mesmo projecto, o match ainda será por `last_active`. A solução definitiva seria cruzar com PID via `lsof`, mas isso já existe como fallback async em `findTerminalByLsof` — que é chamado quando os dois primeiros falham.
