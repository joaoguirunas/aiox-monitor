# Story 8.4 — Session-Aware PID Deduplication in syncSystemTerminals

## Story

**Como** sistema de monitorização,
**Quero** que `syncSystemTerminals` verifique se a sessão Claude Code já está rastreada antes de criar um novo terminal,
**Para que** a mesma sessão não gere registos duplicados com PIDs diferentes.

## Context

**Bug actual:** O hook envia PID via `os.getppid()` (PID do processo pai Claude Code). O system detection (`syncSystemTerminals`) detecta PIDs via `ps -t TTY | grep claude | head -1`. Estes PIDs podem diferir — o hook pega o parent, a detection pode pegar um child process. Resultado: dois registos DB para a mesma sessão. A constraint `UNIQUE(project_id, pid)` não previne isto porque os PIDs são diferentes.

**Referência:** `docs/Terminais.md` secção 5.2 | Epic 8 Bug Ref 5.2

## Dev Notes

### Obter PGID
```bash
ps -o pgid= -p 12345  # retorna o PGID do processo
```
Processos no mesmo grupo (parent + children) partilham o PGID. O hook captura o parent PID; se a system detection captura um child, ambos terão o mesmo PGID.

### Ordem de verificação no loop de criação
```typescript
for (const [pid, info] of detectedByPid) {
  if (dbPidSet.has(pid)) continue;
  if (!info.isClaudeProcess) continue;

  // 1. Check session_id (rápido, sem fork)
  if (info.sessionId) {
    const exists = db.prepare(
      `SELECT 1 FROM terminals WHERE session_id = ? AND status != 'inactive' LIMIT 1`
    ).get(info.sessionId);
    if (exists) continue;
  }

  // 2. Check PGID (fork ps, mais lento)
  const pgid = runSilent(`ps -o pgid= -p ${pid}`).trim();
  if (pgid) {
    const sameGroup = dbTerminals.some(t => {
      if (t.status === 'inactive') return false;
      const tPgid = runSilent(`ps -o pgid= -p ${t.pid}`).trim();
      return tPgid === pgid && tPgid !== '';
    });
    if (sameGroup) continue;
  }

  // ... criar terminal normalmente
}
```

### Performance
- `ps -o pgid= -p PID` é O(1) — leitura do procfs, ~1ms
- No worst case com 10 terminais activos: 10 x 1ms = 10ms por ciclo sync (a cada 30s) — negligível
