# Story 8.5 — Session-Aware Upsert — Reset on Session Change

## Story

**Como** sistema de monitorização,
**Quero** que `upsertTerminal` resete campos de sessão quando o `session_id` muda num conflito de PID,
**Para que** o PID recycling pelo OS não contamine dados entre sessões diferentes.

## Context

**Bug actual:** A `upsertTerminal` usa `COALESCE(excluded.X, X)` em todos os campos no `ON CONFLICT`. Quando o OS recicla um PID (processo A morre, processo B obtém o mesmo PID), o registo do processo A é actualizado mas mantém dados stale:
- `first_seen_at` da sessão anterior (não reflecte quando a nova começou)
- `agent_name` da sessão anterior (se a nova não envia agente)
- `current_tool` / `current_input` da sessão anterior

**Referência:** `docs/Terminais.md` secção 5.5 | Epic 8 Bug Ref 5.5

## Acceptance Criteria

### Task 1: Session-aware ON CONFLICT
- [x] **AC1** — Em `upsertTerminal()` (`src/lib/queries.ts`), detectar mudança de sessão: quando `excluded.session_id IS NOT NULL AND excluded.session_id != COALESCE(session_id, '')`, fazer reset completo dos campos
- [x] **AC2** — Campos resetados quando sessão muda:
  - `session_id` = novo valor
  - `agent_name` = novo valor (não COALESCE — pode ser NULL)
  - `agent_display_name` = novo valor (não COALESCE)
  - `current_tool` = novo valor (não COALESCE)
  - `current_input` = novo valor (não COALESCE)
  - `current_tool_detail` = NULL
  - `waiting_permission` = 0
  - `first_seen_at` = `datetime('now')` (reset timestamp)
- [x] **AC3** — Quando a sessão NÃO muda (mesmo session_id ou session_id NULL), manter o comportamento actual com COALESCE

### Task 2: Validação
- [ ] **AC4** — Fechar Claude Code (Stop event → terminal inactivo). Reabrir no mesmo terminal (pode obter mesmo PID). Verificar que `first_seen_at` reflecte o novo início e `agent_name` não herda da sessão anterior
- [ ] **AC5** — Hook normal sem mudança de sessão continua a funcionar (COALESCE preserva dados parciais)
- [x] **AC6** — `npm run build` passa sem erros

## Dev Notes

### SQL com detecção de sessão
```sql
ON CONFLICT(project_id, pid) DO UPDATE SET
  session_id         = COALESCE(excluded.session_id, session_id),
  status             = 'processing',
  agent_name         = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN excluded.agent_name
    ELSE COALESCE(excluded.agent_name, terminals.agent_name)
  END,
  agent_display_name = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN excluded.agent_display_name
    ELSE COALESCE(excluded.agent_display_name, terminals.agent_display_name)
  END,
  current_tool       = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN excluded.current_tool
    ELSE COALESCE(excluded.current_tool, terminals.current_tool)
  END,
  current_input      = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN excluded.current_input
    ELSE COALESCE(excluded.current_input, terminals.current_input)
  END,
  window_title       = COALESCE(excluded.window_title, terminals.window_title),
  current_tool_detail = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN NULL
    ELSE terminals.current_tool_detail
  END,
  waiting_permission = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN 0
    ELSE terminals.waiting_permission
  END,
  first_seen_at      = CASE
    WHEN excluded.session_id IS NOT NULL
     AND excluded.session_id != COALESCE(terminals.session_id, '')
    THEN datetime('now')
    ELSE terminals.first_seen_at
  END,
  last_active        = datetime('now')
```

**Nota:** `window_title` mantém COALESCE — o título do terminal não muda com a sessão Claude Code.
