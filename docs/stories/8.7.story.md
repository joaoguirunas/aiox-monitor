# Story 8.7 — Terminal Health Observability Endpoint

## Story

**Como** operador do aiox-monitor,
**Quero** um endpoint `/api/terminals/health` que reporte métricas de saúde do módulo Terminais,
**Para que** problemas de deduplicação, matching e enrichment sejam detectáveis sem inspecção manual da DB.

## Context

**Gap actual:** Não há forma de saber se o JSONL watcher está a fazer match correcto, quantos terminais estão órfãos, ou qual a taxa de enrichment. Diagnosticar problemas requer queries manuais na DB SQLite.

**Referência:** `docs/Terminais.md` secção 5.10 | Epic 8 SC-7

## Acceptance Criteria

### Task 1: Queries de auditoria
- [ ] **AC1** — Adicionar em `src/lib/queries.ts` uma função `getTerminalHealth()` que retorna:
  ```typescript
  interface TerminalHealth {
    total: number;              // Total de terminais na DB
    active: number;             // status = 'processing' ou 'active'
    inactive: number;           // status = 'inactive'
    duplicateSessions: number;  // session_ids com >1 terminal (SC-1)
    orphanTerminals: number;    // terminais activos sem session_id (matching falhou)
    enrichedCount: number;      // terminais activos com current_tool_detail != NULL
    enrichmentRate: number;     // enrichedCount / active (0-1, float)
    waitingPermission: number;  // terminais com waiting_permission = 1
  }
  ```
- [ ] **AC2** — Query de duplicados: `SELECT COUNT(*) FROM (SELECT session_id FROM terminals WHERE session_id IS NOT NULL AND status != 'inactive' GROUP BY session_id HAVING COUNT(*) > 1)`
- [ ] **AC3** — Query de órfãos: `SELECT COUNT(*) FROM terminals WHERE session_id IS NULL AND status != 'inactive'`

### Task 2: Endpoint API
- [ ] **AC4** — Novo route `src/app/api/terminals/health/route.ts` com handler `GET`
- [ ] **AC5** — Endpoint retorna JSON com a estrutura `TerminalHealth` e status 200
- [ ] **AC6** — Endpoint é `force-dynamic` (sem cache)

### Task 3: Validação
- [ ] **AC7** — `GET /api/terminals/health` retorna resposta válida com todas as métricas
- [ ] **AC8** — Após correcções de 8.1-8.5, `duplicateSessions` = 0 e `enrichmentRate` > 0 quando há terminais activos
- [ ] **AC9** — `npm run build` passa sem erros
