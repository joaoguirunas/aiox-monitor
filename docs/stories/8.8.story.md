# Story 8.8 — Unit Tests for Terminal Matching & Lifecycle

## Story

**Como** equipa de desenvolvimento,
**Quero** testes unitários que cubram path conversion, PID dedup, session-aware upsert, e cleanup lifecycle,
**Para que** regressões nos 6 bugs corrigidos nesta epic sejam detectadas automaticamente.

## Context

**Gap actual:** Nenhum ficheiro do módulo Terminais tem testes. A complexidade da lógica de matching (3 estratégias de fallback), lifecycle (4 transições de status), e deduplicação (session_id + PGID) torna a ausência de testes arriscada — qualquer alteração futura pode re-introduzir os bugs.

**Pré-requisito:** Stories 8.1–8.5 devem estar completas (os testes validam o comportamento corrigido).

**Referência:** `docs/Terminais.md` secção 5.8 | Epic 8 SC-1 a SC-6

### DB in-memory para testes
```typescript
import { DatabaseSync } from 'node:sqlite';
import { initSchema } from '../src/lib/schema';

export function createTestDb(): DatabaseSync {
  const db = new DatabaseSync(':memory:');
  initSchema(db);
  return db;
}
```

**Nota:** As funções em `queries.ts` usam um singleton `db` importado de `db.ts`. Para testes, será necessário injectar a instância ou usar mock. A abordagem recomendada é extrair as queries como funções puras que recebem `db` como parâmetro, ou usar jest.mock/vitest.mock no módulo `db.ts`.

### Cobertura alvo
Os testes devem cobrir exactamente os 6 cenários de bug corrigidos:
1. Path com hífen → match correcto (8.1)
2. Terminal sem projecto → ignorado (8.2)
3. Múltiplos terminais → prioridade processing (8.3)
4. Mesmo PGID/session_id → dedup (8.4)
5. Session change → reset campos (8.5)
6. Cleanup lifecycle → transições correctas
