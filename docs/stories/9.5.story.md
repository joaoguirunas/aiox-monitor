# Story 9.5 — Busca Textual Server-Side

## Story

**Como** utilizador do aiox-monitor,
**Quero** pesquisar por texto em prompts, respostas e comandos de tools,
**Para que** consiga encontrar rapidamente uma acção ou conversa específica no histórico.

## Context

A FilterBar actual permite filtrar por projecto, agente, terminal e tipo de evento, mas não por conteúdo textual. Com centenas de eventos, encontrar uma interacção específica requer scroll manual. A busca textual via SQL LIKE é suficiente para os volumes actuais (< 10k eventos).

## Dev Notes

### SQL para busca em sessões

```sql
-- Sessões onde algum evento contém o texto
WHERE EXISTS (
  SELECT 1 FROM events e
  WHERE e.session_id = s.id
  AND (e.input_summary LIKE ? OR e.output_summary LIKE ?)
)
```

### Performance

SQLite LIKE com `%texto%` não usa index — full scan. Aceitável para < 10k eventos. Se necessário no futuro, migrar para FTS5.

### Tipos

Adicionar `search?: string` a `EventFilters` e `SessionFilters`.
