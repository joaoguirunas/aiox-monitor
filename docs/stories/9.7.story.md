# Story 9.7 — Testes, Cleanup & Export

## Story

**Como** developer do aiox-monitor,
**Quero** testes unitários para os módulos novos da Lista, fix do status `interrupted`, e export de dados,
**Para que** o módulo seja robusto, testado e o utilizador possa extrair dados quando necessário.

## Context

A Epic 9 adiciona vários módulos novos (`getSessions`, `useSessions`, paginação em `useEvents`) que não têm cobertura de testes. O status `interrupted` está no schema mas nunca é atribuído. O utilizador não consegue exportar dados da Lista.

## Dev Notes

### Status interrupted

Adicionar ao cleanup scheduler em `server.ts` (junto com `cleanupOldEvents`):

```typescript
// Mark stale active sessions as interrupted
db.prepare(`
  UPDATE sessions SET status = 'interrupted'
  WHERE status = 'active'
  AND id NOT IN (
    SELECT DISTINCT session_id FROM events
    WHERE created_at > datetime('now', '-1 hour')
    AND session_id IS NOT NULL
  )
`).run();
```

### Export format

```json
{
  "exported_at": "2026-03-18T20:00:00Z",

}
```
